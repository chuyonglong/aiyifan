# 搜索目录回退设计

## 背景

`SearchActivity` 已接入 `RemoteCatalogRepository`。线上配置和联想接口可用，但
`api/Search/GetSearch` 在 2026-07-24 返回 HTTP 200、业务码 `ret=202` 与空数据。
现有代码把该错误无提示地降级到 mock 数据，因而真实片名无法搜索。

现有产品计划已将搜索纳入 V1 第 5 阶段。本次只补齐“视频结果可搜索”的主链路，
不增加用户、动态、相册搜索或服务端分页。

## 方案选择

1. 仅修正远程接口路径或参数：原 APK 使用过 `SearchCriteria`，但当前服务使用该参数
   仍返回 `ret=202`，不能恢复功能。
2. 仅展示远程错误：能如实反映问题，但用户无法搜索。
3. 使用已加载的真实首页目录作为回退索引：不依赖失效的上游搜索服务，并且结果可直接
   进入现有播放流程。

采用方案 3。远程搜索成功时仍优先使用远程结果；只有网络、HTTP、业务码或解析失败时，
才搜索真实首页目录。mock 数据只在真实目录不可获得时作为最后回退。

## 架构与数据流

`RemoteCatalogRepository.searchVideos(keyword)` 先请求远程接口，并将 HTTP 状态、业务码和
可解析的数据作为成功条件。若失败或空结果，则调用 `ensureSections()` 获取或复用首页目录，
对去重后的 `VideoSummary` 执行本地匹配。匹配字段为标题、内容类型、地区、年份、演员和导演。

首页接口内的 `mediaType`（例如“电影”）也要映射进可搜索字段，确保热门词“电影”能匹配
真实目录中的电影条目。结果沿用现有 `VideoSummary` 和 `SearchResultAdapter`，点击行为不变。

`SearchActivity` 不把“远程失败但目录回退成功”显示为“搜索失败”；只显示结果数量。真正没有
匹配项显示空结果。当前没有可观测的失败状态，本次不新增 UI 或埋点，以保持改动范围最小。

## 错误处理

- HTTP 非 2xx、非 200 业务码、无效 JSON 或无法提取列表：触发目录回退。
- 远程返回空列表：也使用目录回退；若目录同样无匹配，再使用 mock 回退。
- 首页目录加载失败：保留现有 mock 回退，确保离线演示环境仍可搜索。
- 空关键词：不发请求，返回空列表，保持现有页面校验。

## 测试

为 `RemoteCatalogRepository` 增加纯单元测试，使用可控 `HttpFetcher`：

- 上游返回 `ret=202` 时，结果来自真实首页目录而非 mock。
- 首页条目的 `mediaType` 可被作为搜索词匹配。
- 上游正常返回且有结果时，优先使用远程结果。
- 目录不可用时，保留 mock 搜索回退。

验证执行 `gradlew.bat test` 与 `gradlew.bat lint`，并用实际接口响应再次确认 `ret=202` 场景走目录回退。
