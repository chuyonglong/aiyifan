# V1 实现路线

## 目标

按视频主链路分阶段复刻，保证每个阶段都能运行、能验证、能继续叠功能。

## 阶段 1：工程骨架

- 新建 Android Kotlin 项目，启用 ViewBinding。
- 配置 Retrofit、OkHttp、Glide、Media3、Room、Coroutines。
- 建立 `core/network`、`core/database`、`core/player`、`feature/*` 包结构。
- 实现 `SplashActivity`、`MainActivity`、首页/热门/我的三个 tab 空页面。

验收：

- App 可安装启动。
- 底部 tab 可切换。
- 无网络时不会崩溃。

## 阶段 2：首页与分类

- 实现分类接口和本地缓存。
- 实现首页顶部栏、分类 tab、RecyclerView 多类型列表。
- 实现 banner、海报卡、信息流卡。
- 实现分类页左右栏。

验收：

- 首页能展示真实或 mock 分类。
- 切换分类能刷新列表。
- 点击视频卡能进入视频页并带上 `mediaKey`。

## 阶段 3：视频详情

- 实现视频详情接口和 `VideoDetail` 模型。
- 实现简介页、详情弹层、选集、推荐列表。
- 实现收藏/点赞/不喜欢按钮状态。
- 评论 tab 先展示列表和分页。

验收：

- 视频页能加载详情。
- 多集视频可切换。
- 详情弹层字段完整。
- 评论列表可分页。

## 阶段 4：播放器

- 用 Media3 封装播放器组件。
- 实现竖屏控制栏、横屏全屏控制栏。
- 实现播放/暂停、进度、倍速、清晰度、语言、字幕。
- 实现播放进度本地保存。

验收：

- 视频可播放。
- 全屏切换不重置进度。
- 切换清晰度/倍速可用。
- 退出再进恢复播放进度。

## 阶段 5：搜索

- 实现搜索历史 Room 表。
- 实现热搜、联想、视频结果列表。
- 实现搜索结果进入播放。

验收：

- 搜索词保存和清空正常。
- 联想和结果页状态切换正常。
- 搜索结果可播放。

## 阶段 6：我的、历史、收藏、登录

- 实现我的页精简布局。
- 实现观看历史列表、删除、清空。
- 实现收藏列表和取消收藏。
- 实现基础登录页和登录态保存。
- 接入未登录拦截。

验收：

- 游客可浏览播放。
- 登录后可收藏。
- 历史和收藏能持久化。

## 阶段 7：视觉还原与收口

- 按 APK 尺寸统一顶部栏、底部栏、卡片比例、字体大小。
- 替换图标，来源使用 iconfont 或 Google Material Symbols Rounded。
- 统一加载、空态、失败重试。
- 清理所有 v1 不做入口：VIP、支付、音乐、上传、扫码、投屏、消息。

验收：

- 主要页面视觉接近原 APK。
- v1 不做入口不会误导用户进入半成品流程。
- 主链路完整：启动、首页、搜索、详情、播放、历史、收藏。

## 后续版本建议

- v1.1：离线下载、投屏、扫码。
- v1.2：短视频、动态、评论发布。
- v1.3：音乐模块。
- v1.4：VIP/付费/支付。
- v1.5：上传中心、相册、用户主页、私信。
